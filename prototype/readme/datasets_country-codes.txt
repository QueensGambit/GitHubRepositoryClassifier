b'Comprehensive country code information, including ISO 3166 codes, ITU dialing\ncodes, ISO 4217 currency codes, and many others. Provided as a [Simple Data\nFormat Data Package](http://dataprotocols.readthedocs.io/en/latest/simple-data-format.html).\n\n## Data\n\nData comes from multiple sources as follows:\n\nCustomary English short names are from\n[Unicode Common Locale Data Repository (CLDR) Project] (https://github.com/unicode-cldr/cldr-localenames-full/blob/master/main/en/territories.json)\nNote: CLDR shorter names "ZZ-alt-short" are used when available\n\nISO 3166 official English and French short names are from\n[United Nations Statistics Division](http://unstats.un.org/unsd/methods/m49/m49.htm)\n\nISO 4217 currency codes are from\n[currency-iso.org](http://www.currency-iso.org/en/home/tables/table-a1.html)\n\nMany other country codes are from\n[statoids.com](http://www.statoids.com/wab.html)\n\nSpecial thanks to Gwillim Law for his excellent\n[statoids.com](http://www.statoids.com) site (some of the field descriptions\nare excerpted from his site), which is more up-to-date than most similar\nresources and is much easier to scrape than multiple Wikipedia pages.\n\nCapital cities, languages, continents, TLDs, and geonameid are from [geonames.org](http://download.geonames.org/export/dump/countryInfo.txt)\n\nEDGAR codes are from [sec.gov](https://www.sec.gov/edgar/searchedgar/edgarstatecodes.htm)\n\n\n## Preparation\n\nThis package includes Python scripts to fetch current country information\nand output a JSON document and CSV of combined country code information.\n\nCSV output is provided via the `in2csv` and `csvcut` utilities from [csvkit](http://github.com/onyxfish/csvkit)\n\n### data/country-codes.csv\n\nInstall requirements:\n\n    pip install -r scripts/requirements.pip\n\n\nRun GNU Make to generate data file:\n\n    make country-codes.csv\n\n## License\n\nThis material is licensed by its maintainers under the Public Domain Dedication\nand License.\n\nNevertheless, it should be noted that this material is ultimately sourced from\nISO and other standards bodies and their rights and licensing policies are somewhat\nunclear. As this is a short, simple database of facts there is a strong argument\nthat no rights can subsist in this collection. However, ISO state on [their\nsite](http://www.iso.org/iso/home/standards/country_codes.htm):\n\n> ISO makes the list of alpha-2 country codes available for internal use and\n> non-commercial purposes free of charge.\n\nThis carries the implication (though not spelled out) that other uses are not\npermitted and that, therefore, there may be rights preventing further general\nuse and reuse.\n\nIf you intended to use these data in a public or commercial product, please\ncheck the original sources for any specific restrictions.\n\n'
