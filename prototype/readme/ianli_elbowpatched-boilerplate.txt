b"# [Elbowpatched Boilerplate](http://elbowpatched.com)\n\nElbowpatched Boilerplate is a template to make it easier to create an academic personal website.\nIt is built using [Jekyll](http://jekyllrb.com/), a simple, blog-aware, static site generator.\nSince this template is built on Jekyll, \nyou can publish your website on [GitHub Pages](https://pages.github.com/) for free.\n\n* Source: [https://github.com/ianli/elbowpatched-boilerplate](https://github.com/ianli/elbowpatched-boilerplate)\n* Homepage: [http://elbowpatched.com](http://elbowpatched.com)\n* Twitter: [@elbowpatched](http://twitter.com/elbowpatched)\n\n\n## Why?\n\nAs a scholar, having an academic personal website can be a good way for you to publicize your research.\nYour website can have descriptions of your research projects, \na list of your published papers, \nand links to your slides and video presentations.\n\nHowever, creating an academic personal website takes a lot of time.\nTime that you also have to spend reading papers, conducting studies, analyzing data, and writing papers.\n\nElbowpatched Boilerplate is here to help!\nIt has the following features that make it easier to create an academic personal website.\n\n* Fill out simple files with information about yourself, your publications, projects, and presentations.\n* Use the provided templates or create your own to change the presentation of your content.\n* Publish your website on GitHub Pages for free.\n\n\n## Quick start\n\n* Clone the git repo - `git clone https://github.com/ianli/elbowpatched-boilerplate.git`.\n* Setup and run [Jekyll](http://jekyllrb.com/).\n* Edit the contents of the [YAML](http://en.wikipedia.org/wiki/YAML) files in the folder `_data`.\n  * `personal.yml` - Your personal information\n  * `publications.yml` - Bibliography of your publications\n  * `projects.yml` - Descriptions of your projects with pictures\n  * `presentations.yml` - Your presentations with links to slides\n* Preview your website at `index.html`\n\n\n## Layouts\n\nElbowpatched Boilerplate currently only has one template that you can use: \n[Albertine](https://github.com/ianli/elbowpatched-boilerplate/blob/gh-pages/_layouts/albertine.html).\nFeel free to modify it to your needs.\nI will add more layouts as this project progresses.\n\nIf you want to create a new template, take a look at \n[Albertine](https://github.com/ianli/elbowpatched-boilerplate/blob/gh-pages/_layouts/albertine.html) \nfor inspiration. \nThe [Templates documentation](http://jekyllrb.com/docs/templates/) on the Jekyll site and\n[Liquid Basics](http://docs.shopify.com/themes/liquid-documentation/basics) \nare good introductions to Liquid, the template language of Jekyll.\n\n\n## Publishing options\n\nWhen you're ready to publish your website, you have two options:\n\n1. Publish on GitHub Pages - [instructions](https://pages.github.com/).\n2. Publish on your own webserver - [instructions](http://jekyllrb.com/docs/deployment-methods/).\n\n\n## License\n\nSee [LICENSE](https://github.com/ianli/elbowpatched-boilerplate/blob/gh-pages/LICENSE)."
