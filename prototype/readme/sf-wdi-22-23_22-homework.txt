b'#SF-WDI-22 Homework submission\n\nHomework will be turned in to your fork of this repository daily.\n\n##Setup\n\nTo set up your initial homework repository do the following:\n\n* fork this repo to create a copy of it on your github account\n* clone the forked repo from your github account onto your computer inside your `dev/` directory\n* `cd` into the newly cloned repo on your computer\n* `mkdir **GIT-USER-NAME**`\n* `cd **GIT-USER-NAME**`\n* `echo "#**YOUR NAME**\'s Homework repo" > README.md` to set up your repo\'s README\n* `git add -A` to stage your changes\n* `git commit -m "inital commit"` to have git save your changes\n* `git push origin master` to put your changes up on github!\n\n**GIT-USER-NAME** should be replaced with your actual git user name (must be exact)\n\n**YOUR NAME** should be your actual name.\n\n\n\n##Submit homework\n\nEach day you will submit your homework by doing the following from within your homework repo on your computer:\n\n* copy any required files into your homework repo directory as instructed in homework writeup\n* `git add -A` to stage all your changes\n* `git commit -m "homework commit"` to have git save the changes -- feel free to add to the commit message if you\'d like\n* `git push origin master` to put the changes up on github\n* go to your homework repo\'s page on your account on github.com\n* create a pull request according to the guidelines below\n\n####Pull request details\n\n(Must be in this format)\n\n**Title:** `w##d##`\n\nReplace ## with week and day number (eg w01d02  = homework for week 1 day 2)\n\n**Message:**\n\n```\ncomfort: #\ncompleteness: #\n\nMessage about your experience goes here\n```\n\nReplace the `#`s next to comfort and completness with a number (1 - 5) to represent how comfortable you are with the topic and how much of the assignment you were able to complete.  A 1 is the lowest (least comfortable / very little complete), and a 5 is the highest (most comfortable / fully complete).\n'
